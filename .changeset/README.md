# 发版流程指南

## 日常使用

### 1. 记录变更（开发后）

当你完成代码修改后，创建一个 changeset：

```bash
pnpm changeset
```

选择受影响的包（空格选择，回车确认）：
- `doc2ts` - 主包
- `doc-pre-data` - 依赖包

然后选择版本类型：
- `patch` - 修复 bug (0.0.x)
- `minor` - 新功能 (0.x.0)
- `major` - 重大变更 (x.0.0)

最后输入变更描述。

### 2. 提交代码

```bash
git add .
git commit -m "feat: xxx 功能"
```

### 3. 发版（发布到 npm）

#### 一键发版（推荐）：
```bash
pnpm publish:all
```

这个命令会：
1. 自动计算并更新版本号
2. 更新 pnpm-lock.yaml
3. 构建所有包
4. 发布到 npm

#### 分步发版：
```bash
# 1. 更新版本号
pnpm changeset:version

# 2. 重新安装依赖（更新 lock 文件）
pnpm install

# 3. 构建并发布
pnpm changeset:publish
```

## 注意事项

1. **linked 配置**：doc2ts 和 doc-pre-data 已设置为 linked，版本会同步更新
2. **workspace 依赖**：doc-pre-data 通过 `catalog:` 协议引用，发版时会自动更新
3. **自动更新依赖**：当 doc-pre-data 发版时，doc2ts 中的依赖版本会自动更新
